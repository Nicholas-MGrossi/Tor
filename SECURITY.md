# Security Policy

## Supported versions
This repository follows best-effort support for the default branch.

## Reporting a vulnerability
If you believe you have found a security issue:
1. Do **not** open a public issue with sensitive details.
2. Contact the maintainers privately (add contact method here).
3. Include steps to reproduce and impact assessment.

## Secure development guidelines
- No secrets in the repository.
- Use pull requests and reviews.
- Prefer dependency pinning and regular updates.
